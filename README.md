## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/yourusername/newsletter-idea-generator.git
cd newsletter-idea-generator
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up Environment Variables

Create a `.env.local` file in the root directory and add the following keys:

```env
# Azure OpenAI Configuration
AZURE_OPENAI_API_KEY=your_azure_api_key
AZURE_OPENAI_ENDPOINT=https://your-resource-name.openai.azure.com
AZURE_OPENAI_DEPLOYMENT=gpt-4
AZURE_OPENAI_API_VERSION=2024-02-15-preview

# Supabase (for Email Gate)
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

> **Note**: If you want to use standard OpenAI (non-Azure), update `lib/openai.ts` to use the standard `OpenAI` client configuration.

### 4. Run the development server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Database Setup (Supabase)

1.  Create a new Supabase project.
2.  Go to the **SQL Editor** and run:
    ```sql
    create table emails (
      id bigint generated by default as identity primary key,
      created_at timestamp with time zone default timezone('utc'::text, now()) not null,
      email text not null unique
    );
    ```
3.  **Enable RLS (Row Level Security)**:
    -   Go to **Authentication > Policies**.
    -   Create a new policy for the `emails` table.
    -   Allowed operation: `INSERT`.
    -   Target roles: `public` / `anon`.
    -   Expression: `true`.

## Deployment

The easiest way to deploy is using **Vercel**.

1.  Push your code to GitHub.
2.  Import the project in Vercel.
3.  Add the Environment Variables from your `.env.local` file.
4.  Deploy!
